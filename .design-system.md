## 8. Math Rendering

**Decision: Standardize on KaTeX** (not MathJax).

| | MathJax | KaTeX |
|-|---------|-------|
| Render speed | ~500ms | ~50ms (10x faster) |
| Bundle size | 1.2MB | 200KB |
| Visual quality | Excellent | Excellent |
| Offline | Needs CDN or big local copy | Smaller local footprint |

Current state: 4/12 files use MathJax, 8 use plain text. Moving to KaTeX everywhere gives fast rendering with clean LaTeX notation for all equations.

```html
<link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/katex@0.16/dist/katex.min.css">
<script src="https://cdn.jsdelivr.net/npm/katex@0.16/dist/katex.min.js"></script>
<script src="https://cdn.jsdelivr.net/npm/katex@0.16/dist/contrib/auto-render.min.js"></script>
```

## 14. Voice & Personality

The copy should sound like a coach, not a textbook. Kai is 15. The tool should respect his intelligence while keeping energy high.

**Do:**
- "You nailed Standard 3.a. Now let's work on 3.c."
- "3 wrong on exponents. Let's run those again."
- "ğŸ”¥ 7 in a row. Keep going."

**Don't:**
- "Great job, buddy!" (patronizing)
- "You need to study more." (discouraging)
- "Please review the following incorrect responses." (robotic)

**The vibe:** Supportive but direct. Like a good coach at halftime â€” acknowledges what's working, points to what needs work, sends you back out with confidence.
