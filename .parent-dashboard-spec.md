## Components

### Latest Test Card

Shows the most recent test result prominently.

- **Background:** Navy gradient (same as passing score card) if â‰¥90%, off-white if <90%
- **Content:** Test name, date, score, time, voice copy
- **Voice copy:** Uses same tiers as `.voice-guide.md` score feedback

### Score Trend Chart

Same as Kai's dashboard (`.chart-spec.md` Â§2) but:
- Shows ALL units (multi-series lines if multiple units have data)
- Legend enabled (unit names as series labels)
- Pistons-blue for current unit, chrome for past units
- Target line at 80%

### Standards Grid

Same component as `.scorecard-spec.md` standard cards:
- 3 columns desktop, 1 column phone
- Sorted lowest score first
- Color tiers: green â‰¥90%, blue 70-89%, amber 50-69%, red <50%
- Voice copy per standard

### Time Spent Chart

- **Type:** Horizontal bar chart
- **X-axis:** Minutes (0 to max)
- **Y-axis:** Test names
- **Bar color:** pistons-blue `#1D42BA`
- **Height:** 150px phone, 200px desktop
- **Purpose:** Marcus can see if Kai is spending enough time (rushing = low scores)

### Summary Stats

Plain text block, no chart:
```
Tests taken: 4
Average score: 72%
Average time: 39 min
Best score: 88% (Feb 17)
Current unit: Exponents
```

- **Style:** Key-value pairs, left-aligned, mid-gray labels, dark-gray values
- **Font:** 0.95em

## Data Source

Reads from `localStorage`:

```javascript
// Same key as Kai's dashboard
const scores = JSON.parse(localStorage.getItem('mcm_scores') || '{}');

// Each entry:
// "mcm-unit2-nonlinear-review": {
//     score: 18, total: 20, pct: 88,
//     timestamp: "2026-02-17T..."
// }
```

No separate data model. Parent and student dashboards read the same data, just present it differently.

## Differences from Kai's Dashboard

| Aspect | Kai (index.html) | Marcus (parent.html) |
|--------|------------------|---------------------|
| Focus | "What to study next" | "How is Kai doing" |
| Chart | Current unit only | All units |
| Time data | Not shown | Prominently shown |
| Voice | Coach ("Run it again") | Observer ("88%. Improving.") |
| Actions | Link to tests | View-only |
| Primary device | Laptop | Phone |

### Parent Voice Copy

The parent dashboard uses a slightly different voice â€” informational, not coaching:

| State | Copy |
|-------|------|

| Flat | "Holding at 72%. Consistent." |
| Declining | "Dropped from 80% to 65%." |

| High mastery | "Strong across the board." |

## Print

Parent dashboard does NOT print. `@media print { body { display: none; } }` â€” there's nothing to put on paper here.

The parent dashboard has no password or separate access control. It's a local HTML file reading localStorage. If Marcus needs data from Kai's machine, Kai exports (Agent S's Feature #3).
